# Security Policy

## Supported Versions

Security updates are provided for the latest stable version on the `main` branch.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Use GitHub Security Advisories:

- Open a private report: `Security` tab -> `Report a vulnerability`
- Or use: `https://github.com/XCO-Agency/Laravel-SAAS-Starter/security/advisories/new`

When reporting, include:

1. A clear description of the issue and impact.
2. Reproduction steps or proof-of-concept.
3. Affected version/commit and environment details.
4. Any suggested remediation (optional).

You should receive an acknowledgment within 3 business days.
